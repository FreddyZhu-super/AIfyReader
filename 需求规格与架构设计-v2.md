# Peter Markdown — 需求规格与架构设计 v2

> 根据用户明确的需求，更新架构设计。核心交互方式确认：**WYSIWYG 原地编辑（Typora 模式）** + **可折叠弹出式大纲（TOC）**。

### 功能明细

**1. 文件夹浏览模式**
- 打开一个文件夹 → 左侧显示文件树（只显示支持的 Markdown 文件）
- 点击文件 → 右侧**只读渲染** Markdown 内容
- 文件树支持展开/折叠子目录
- 文件树可按文件名搜索过滤

**2. 单文件编辑模式**
- 默认进入时，显示该文件的 WYSIWYG 渲染视图
- 在渲染视图上**直接编辑**：点击文字可修改，输入 Markdown 语法自动转换
- 输入 `**text**` → 立即变成粗体，输入 `# ` → 立即变成标题
- 代码块、表格、图片等复杂元素都有对应的编辑交互
- 支持 Ctrl+Z / Ctrl+Shift+Z 撤销/重做

**3. Header 大纲（TOC）**
- 点击工具栏按钮或按快捷键（如 Ctrl+Shift+O）弹出大纲面板
- 大纲解析当前文件的所有标题（h1-h6），按层级缩进展示
- 点击大纲条目 → 编辑器滚动到对应标题位置并高亮
- 大纲关闭后自动隐藏

**4. 文件支持**
- 支持的扩展名：`.md` `.mdx` `.markdown` `.mdown` `.mkd`
- 文件编码：UTF-8（自动检测 BOM）

## 四、大纲 TOC 模块详细设计

### 数据流

```
Markdown 内容 (ProseMirror Document)
         │
         ▼
  [大纲解析器]
    提取所有 heading 节点 (h1-h6)
         │
         ▼
  构建层级树:
  ─ H1 ─ H2 ─ H3 ─ H2 ─ H3 ─ H3 ─ H2 ...
         │
         ▼
  渲染为可点击的大纲 UI
         │
         ▼
  用户点击 → 计算 heading 位置 → editor.dispatch(scrollIntoView)
                                       → 高亮该 heading
```

### 大纲解析规则

```typescript
interface TocItem {
  level: 1 | 2 | 3 | 4 | 5 | 6
  text: string           // 标题纯文本（不含 Markdown 标记）
  anchor: string        // 锚点 ID（用于跳转）
  position: number      // 在文档中的字符偏移/节点位置
  children: TocItem[]   // 子标题
}
```

- 缺失中间层级时自动缩进（如 h1 → h3 跳级时，h3 作为 h1 的子级缩进）
- 相同层级依次排列
- 锚点 ID 生成规则：中文标题用拼音或数字索引，英文标题 slugify

## 五、用户交互流程

### 流程 1：打开文件夹浏览

```
启动 App → 点击 "Open Folder" → 选择目录
         → 左侧文件树加载所有 .md/.mdx/... 文件
         → 右侧显示空白/欢迎页
         → 点击文件 → 右侧渲染内容（只读模式）
         → 双击文件 → 切换到单文件编辑模式
```

### 流程 2：编辑单个文件

```
打开文件（编辑模式）
         → 编辑器进入 WYSIWYG 模式
         → 用户输入 `## 新标题`
         → 输入规则触发：自动转换为 h2 标题
         → 内容实时渲染
         → Ctrl+S 保存
         → 文件写入磁盘
```

### 流程 3：使用大纲导航

```
正在编辑长文档
         → 点击工具栏大纲图标 / 按 Ctrl+Shift+O
         → 大纲面板弹出（覆盖层/侧边栏）
         → 显示标题层级树
         → 点击 "第三章"
         → 编辑器滚动到第三章位置
         → 第三章标题短暂高亮
         → 大纲面板自动关闭（或保持打开，由配置决定）
```

## 六、对原有架构的影响

### 与原架构的主要变化

| 项目 | 原计划 | 更新后 |
|------|--------|--------|
| 编辑方式 | ProseMirror WYSIWYG | ✅ 保留，确认采用 |
| 预览模式 | 分栏/切换 | ✅ 单一 WYSIWYG 视图 |
| 大纲 | 左侧面板 | 改为**可折叠弹出式** |
| 文件扩展名 | 仅 .md | 扩展为 5 种 |
| 文件夹浏览 | 仅有文件树 | 新增**只读渲染模式** |

### 新增的测试需求

基于这些更新，需要新增以下测试：

1. **大纲/TOC 模块**
   - 从 ProseMirror document 中正确提取标题
   - 构建正确的层级树
   - 跳转锚点计算
   - 中英文标题的锚点生成

2. **文件夹浏览模式**
   - 文件树只显示支持的扩展名
   - 点击文件只读渲染
   - 双击切换到编辑模式

3. **弹出式 UI**
   - 大纲面板的打开/关闭
   - 快捷键触发
   - 点击后自动关闭行为

4. **多扩展名支持**
   - .mdx / .markdown / .mdown / .mkd 文件的正确识别和处理
